程序外接接口是：
API_Interface.h	 &&	API_Interface.cpp


（大部分沿用张琪代码，更改少量！）


API_Interface.cpp  ::

GlobalInit() ::	加载数据库

注意事项：
数据库分为 DenseSurf && DenSurfProVlad  两种，分别存储在工程源目录的上一层目录（采用相对路径）
但是其中  imgPath.txt 中保存的检索库中图片路径使用的是  绝对路径 


Retrieval(Mat img) :: 计算图片检索结果

注意事项：
检索结果存储于全局变量 IRResult ，所有的全局变量均定义在 Configure (.h 或.cpp)中，
附带计算query图片的proposal结果并存储在全局变量 QueryPro 中！

QueryRe 是一个结构体：
PathDis	存储检索得到库图片路径+对应相似度
IdxQuery 匹配策略中query图片最近的proposal的索引（用于MFC显示）
IdxDB	匹配策略中库图片最近的proposal的索引	（用于MFC显示）


ShowToMFC() :: 得到MFC需要的图片和相似度

注意事项：
此函数仅供参考，可根据你MFC设计及自己编程习惯更改

根据之前计算得到的索引结果，计算MFC需要的图片和相似度

存储在全局变量 SimRe && ShowImgs中
其中 根据之前定义的全局变量 feaflags == DENSESURF  或  DENSESURF_PRO_VLAD，来判断MFC的显示

DENSURF需要显示9张图片，分别存储在SimRe && ShowImgs，在翻页的时候需要将 全局变量 pageIdx相应 +1 或者 -1 (初始值是1）

DENSESURF_PRO_VLAD需要显示3张检索库图片 其余是proposal结果，SimRe只有3个值，分别对应第 1 ，4， 7张图片的下方文本框

所有的ShowImgs都是从左到右，从上到下的顺序，对应的显示图片！
